PLATFORM SPECIFICATION
Adaptive Python Learning Platform with AI Feedback
Version: 1.0
________________________________________
1. Product Overview
Product Type
Adaptive online platform for learning Python programming with integrated AI feedback and teacher control system.
The platform has two primary roles:
•	Student — learns Python through structured curriculum and AI-guided feedback
•	Teacher — creates courses, assigns tasks, and monitors student progress
The platform uses OpenAI API to provide contextual, guided feedback instead of direct answers.
________________________________________
2. Core Platform Concept
The platform combines:
•	LMS (Learning Management System)
•	Interactive coding environment
•	Adaptive learning engine
•	AI-powered feedback system
Main principle:
Students solve tasks in a built-in code editor, and AI helps guide their thinking without giving full solutions.
________________________________________
3. User Roles
Role 1 — Student
Student can:
•	Register and log in
•	Select level (Beginner / Intermediate / Advanced)
•	Complete lessons and tasks
•	Write and run Python code
•	Request AI feedback
•	Track progress
•	Complete projects (Boss Challenges)
________________________________________
Role 2 — Teacher
Teacher can:
•	Create courses
•	Create modules, lessons, and tasks
•	Assign tasks to classes
•	Monitor student progress
•	View analytics

6. AI Integration (OpenAI API)
This is the most important part.
AI will be used in 4 main places.
________________________________________
6.1 AI Feedback System (Core Feature)
Location:
Practice Page (/student/practice/:taskId)
Student sees:
Code Editor
Below editor:
"Ask AI for feedback"
Student writes message like:
"I don't understand why my loop doesn't work"
System sends request to backend:
POST /ai-feedback
Payload:
•	student_code
•	student_prompt
•	task_description
•	expected_behavior
•	student_level
•	module_name
•	lesson_name
Backend constructs OpenAI prompt.
Example system prompt:
"You are a Python tutor. Do NOT give full solutions. Guide the student with hints and explanations. Help them understand mistakes."
User message:
Student code + student question + task context
OpenAI returns:
•	feedback
•	hints
•	explanation
This is returned to frontend and displayed.
IMPORTANT RULE:
AI must NOT give full solution code.
Only hints, explanation, debugging guidance.

6.2 AI Debugging Assistance
If student submits incorrect code:
Backend detects failure.
System automatically sends code + error to OpenAI:
Prompt:
"Explain this error and guide student to fix it without giving full solution."
AI response shown to student.
________________________________________
6.3 AI Placement Test Evaluation
During screening tests:
Student code is evaluated.
OpenAI can analyze:
•	code quality
•	logic correctness
•	approach
And assign score.
________________________________________
6.4 AI Hint Generation (optional)
When student clicks "Hint":
System sends:
•	task description
•	student code
OpenAI generates hint.
